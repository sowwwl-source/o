Manifesto layer

Three commitments

Clarity before scale

  1. 01One host, one role. Public copy, product, API, and experiments each keep their own door.
  2. 02Infrastructure should stay reversible. Static pages remain light. Dynamic services remain explicit.
  3. 03Every landing layer should orient the visitor instead of trapping them in ambiguity.

Method

The project gets stronger when durable surfaces and volatile ones stop sharing the same room. Editorial work lives on content domains, application logic lives on the product domain, the API gets its own host, and experiments stay disposable.

A network becomes legible when each door names its purpose.
