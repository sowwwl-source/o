Three commitments

This page should not try to be the product. It should explain the structure around it.

Clarity before scale

  1. One host, one role. Public copy, product, API, and experiments each get their own domain.
  2. Infrastructure should be reversible. Static pages stay static. Dynamic services stay explicit.
  3. The landing layer must tell visitors where to go next, not trap them in ambiguity.

Method

The project grows by separating durable surfaces from volatile ones. That means: editorial work on the content domains, application logic on the product domain, an API on its own host, and experiments on disposable doors.

A network becomes legible when each door names its purpose.
