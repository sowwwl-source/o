Ops notes

Run the stack from the repository root with the production compose file under deploy/.

Deploy

Use a single VPS, point DNS at it, keep Cloudflare in front, and promote only validated structure from sowwwl.org to the hub.

  • Copy deploy/.env.production.example to deploy/.env.production.
  • Set strong database passwords and a real AZA_API_TOKEN.
  • Launch with docker compose --env-file deploy/.env.production -f deploy/docker-compose.prod.yml up --build -d.

DNS

Every domain in this stack should point to the same VPS origin. Add the apex records plus the subdomains below.

  • sowwwl.cloud, 0wlslw0.com, sowwwl.org, sowwwl.com
  • api.sowwwl.cloud, 0.user.o.sowwwl.cloud
  • Redirect www variants at the proxy level, not in DNS only.

SOWWWL Cloud is the canonical control layer after validation. Keep each host single-purpose.
